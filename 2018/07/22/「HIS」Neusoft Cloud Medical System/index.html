「HIS」Neusoft Cloud Medical System

Posted on 2018-07-22 Edited on 2021-11-01

Overview

The system developed in this practice project is the outpatient module in Neusoft Cloud Hospital Management System, which includes the following business functions.

a. Patients: registration and refund function.

b. Outpatient doctor: consultation and prescription function.

c. Chargers: item charging and dispensing function.

My work : front-end design and layout of the page using H-UI components, the use of JavaScript language, jquery components and ajax technology to achieve the local refresh of each display unit; the overall use of SSM framework to achieve back-end logic business and achieve interaction with the database.

System Implementation








Relevant Technologies

Theoretical basis

Maven project build: dependency management tools and project management tools to help developers automatically download artifacts. Solve the version conflict, inconsistency, dependency bloat problem.

SSM main framework: the entire system is divided into view layer, controller layer, service layer, DAO layer. spring MVC is responsible for request forwarding and view management; spring to achieve business object management; mybatis as the data object persistence engine. Its lightweight configuration, the annotation development to the extreme.

Ajax : so that the page does not need to reload, to achieve the page local update technology.

fastjson : third-party tools to complete the mutual transformation of objects and Json strings.

c3p0 connection pooling technology : it is responsible for allocating, managing and releasing database connections to avoid frequent creation and release of connections caused by a large amount of performance overhead, but also to increase the smoothness of the system operating environment; release the idle time exceeds the maximum idle time of the database connection to avoid the database connection caused by the failure to release the database connection omission.

Filter : interception of requests and responses to target resources, so that the system can properly handle Chinese characters.

interceptor : targeted interception of resources, will not meet the requirements of the request and the corresponding interception. In the project to achieve the user must login requirements.

Project development method

The development of the project adopts an iterative-incremental life cycle model, in which each parent business function is subdivided into multiple sub-functional modules during the development process, and each sub-functional module is implemented step by step. Each module is tested individually after implementation, and then integrated with the developed modules of the business function.

When there is a strong correlation between the functions implemented by multiple parent functional modules, these modules will be tested for integration.

Finally, all the business functions will be tested for integration.

At the same time, in the process of development, when it is found that the requirements design is too ideal or does not match the actual requirements, the page will be modified and the corresponding way to achieve the requirements, the sub-functional module code will be changed, and unit and integration testing will be conducted.

Development tools: Eclipse EE

System Analysis

Login

  1. Business Analysis

    When the user is not logged in, the user cannot access other pages under the project; the user enters the account and password, and if the input is correct, the user enters the page corresponding to the authority held by the account, and vice versa, the user is prompted with the wrong account or password input.

  2. Functional analysis

    Set the interceptor, when the user wants to jump to other pages, check whether the user attribute exists in its Session, if it does and is not empty, it will be released, and vice versa, it will jump to the login page.

    Get the account number and password entered by the user, search in the database, enter the account number entered by them exists in the database and the password is correct then jump to the page corresponding to all the privileges of the account; if the account exists but the password is wrongly entered, the password is wrongly prompted; if the account does not exist, the account is wrongly entered.

Registration

  1. Business Analysis

    After the patient provides the operator with personal information such as medical record number or name, ID number, gender, age, birthday, address, etc., and informs the operator of the date, registration level, registered department and corresponding doctor; the operator informs the patient whether the information is reasonable, and if it is not, the patient needs to correct the information; after the patient pays the fee, the registered invoice is printed. After registration and no refund, the patient can go to the doctor’s office for consultation, otherwise the doctor cannot view his personal information.

  2. Functional analysis
    Automatically generate and display invoice number.

    If the patient has registered medical record, when the operator enters the medical record number, all kinds of personal basic information of the patient will be displayed in the corresponding text input box; on the contrary, the operator can generate a new medical record number for the patient and fill in the corresponding information in the personal information field to perfect it.

    Patients can only register the number after the day, meanwhile, as the options of date, department, doctor and registration level change, the options in each input box will make corresponding changes in time. For example, when switching the department or number, the list of doctors will be updated.

    When the registration number is 0, no registration will be given.

    Use JS to verify that the required information in the form is completely filled in, then the form will be submitted; on the contrary, an error message will pop up and the form will not be submitted.

Return number

  1. Business analysis

    When the patient wishes to refund the number, inform the operator of the specific items to be refunded and return the corresponding amount to the patient after completion.

  2. Function analysis

    Enter the medical record number and display all the registered information of the patient for the day.

    Only the registered items in the status of “not seen” can perform the refund operation, the rest of the status are prompted “no refund operation is available”.

Consultation

  1. Business analysis

    When a doctor sees a patient, he/she can fill in the description information to the “Medical Record Home” according to the patient’s own situation, including: chief complaint, current medical history, treatment of current disease, past history, allergy history, physical examination; the doctor can view the queue of patients to be seen and the queue of patients already seen at this time, and can view their personal information and make a choice of the next patient.

  2. Functional analysis
    Use JSTL loop to display the labeled patients to be seen and patients already seen in the form of a list.

    Click on a patient from the list to fill in the medical history information for him/her; in order to select a patient, use JS to check, pop up a prompt message, cannot fill in the medical history, and make all input fields read-only.

    After the patient to be seen, it will be displayed in the list of patients who have been seen until the end of the consultation.

    If the patient selected is a patient who has been seen, the various information previously filled in for him/her will be displayed in the case home page after he/she has been selected.

    Prescription

  3. Business analysis

    Doctors can input keywords from drug items to search and select relevant drugs, or change the search type to template and search for template. After selecting drugs or template, the corresponding drugs will be listed in the prescription column, and doctors can modify the number of drugs in the prescription column as well as add or delete items. You cannot prescribe without items. After clicking on the prescription, the patient can go to the payment counter and pay for it.

  4. Functional Analysis

    The doctor can select the search type, drug item or template item in the search area.

    According to the search category, the fuzzy query will be made for it according to the keyword and displayed in the form of a list.

    After the doctor clicks on the item, the drug corresponding to the item will be added to the prescription column, and the doctor can add or subtract from it, and also modify the quantity of the drug.

Charging

  1. Business Analysis

    According to the medical record number provided by the patient, the operator searches for the items to be paid by the patient, and the patient selects the items he/she wants to pay and makes payment, and cannot pay without items.

  2. Functional analysis

    According to the medical record number provided by the patient, the patient’s personal information will be displayed on the page for verification.

    By clicking on the query, all the unpaid items of the patient will be listed in the form of a JSTL loop to be realized.

    Click on the selection box to select or cancel a single item, and click on the select all box to select all items in the item list.

    Click “Charge” to calculate the amount of all the selected items and bring up the check page, enter the amount of the patient’s payment and calculate the change, click to confirm, and save the payment record to the database.

Dispensing

  1. Business Analysis The pharmacy operator searches according to the medical record number provided by the patient, checks the patient’s desirable medication, and confirms it and gives it out.
  2. Function analysis Based on the medical record number, search and display the paid but uncollected drugs, and check them for distribution. Unpaid and refunded medications will not be displayed.

System Design

Overall Design

  1. System architecture design

    a) the use of Maven deployment SSM framework for managing all the Jar package tools , borrowed from the standard set of directory structure provided to facilitate management ; where the framework is integrated by Spring, SpringMVC, MyBatis open source framework .

    b) Spring provides declarative things , support for the DAO layer to simplify development , as a dependency injection container and AOP implementation exists .

    c) SpringMVC separates the business logic , data and interface display , respectively, to accept user input and call the model and view to complete the user requirements , all data processing work and display the user’s work with the interface for data interaction .

    d) MyBatis encapsulates jdbc, making data more secure and simplifying data access; establishing mapping relationships between entity classes and SQL statements, simplifying code complexity and speeding up development efficiency.

  2. Project directory structure design

    a. Project file directory

    The directory structure is consistent with a standard set of directory structure provided by Maven.

    1. src/main/java: store business logic classes, business logic implementation classes, entity classes, interceptor classes, data access classes, business data classes respectively.

    2. src/main/resources: store the project configuration files about spring, mybatis, and mybatis database access xml files.

    3. src/main/webapp/resource: store front-end pages and front-end static resource files, javascript, css, images, etc. are in this directory.

    4. src/main/webapp/WEB-INF: store all jsp files, and web.xml project files.

    5. src/test/java: store the test class.

    6. pom.xml: store and manage the dependency library and the version information of the dependency library.

    b. Package directory
    PO entity class–package name: com.his.entity; role: store the entity class with the same structure as the database table.

    VO entity class - package name: com.his.dto; role: the entity class that stores the data required by the business logic.

    Data access layer - package name: com. his.dao; role: store the data access layer related interface corresponding xml mapping file.

    Mapper file - path: src/main/resources/mapper; role: store the database access xml file.

    Business logic layer - package name: com. his.service; role: store the interface and implementation classes for the concrete implementation of business logic.

    control layer - package name: com. his.controller; role: to store the front-end request processing, by calling the service interface, to implement a series of business logic controller class.

Conclusion

Through the practice of this course design project, I understood the advantages and the respective roles of the single instance pattern, class diagram, and the high cohesion and low coupling knowledge introduced in the book in the actual development; at the same time, I got involved in the more popular development methods in javaWeb programming, such as SSM framework, Maven project deployment mode, and the application of front-end components. The application of front-end components. I saw the charm of back-end standardized programming, the clear division of labor modules, the role of clear interfaces; front-end HTML table formatting neat layout, in the role of JS and CSS to become friendly and aesthetic and interactive. All these experiences made me experience the process of developing a useful and usable application, and I felt that I was really creating useful values, which made me interested in WEB programming.

In the process of experimentation, I really practiced my ability to solve problems and find information on my own. From ignorance of the concept of the framework, to reading a variety of online materials gradually to its role in development, such as maven’s standardized directory structure, a clear division of the discharge of different functional modules; Spring’s annotated programming, so that development does not focus on the native underlying code, focusing on how to achieve business logic, are greatly accelerated development efficiency.

Similarly, in the practice process, the learning costs faced in turn were too large, taking up time that should have been spent on development; and the learning process and development process go hand in hand, with the quality of each word on the Internet varying plus the iteration of versions, making the learning system confusing from time to time, resulting in a lot of unnecessary time wasted on programming, as well as the fluctuations in mindset. At the same time, when writing the report, I realized that I was lacking in the use of SSM framework and the utilization of front-end components, and I still needed to keep learning. But the problems I encountered in the development, I let how to respond to the error report become more comfortable, but also let me feel that for new things to try, do not stick to the technology they already know and stagnate, try to understand the new technology new framework, its exercise is not only our technology, but also exercise when we are delivered new things, how to learn with a smooth mind, effective means to solve the problem. The ability.

For this practice, my deepest understanding is that it is undesirable to overly study the theoretical implementation of new technologies and ignore the pain point of time constraint when giving them a quick application. After the initial understanding, you can try to use its technology for development, and gradually to achieve the needs, not only is it fast to get started, but also in the paper, the techniques and statements learned in the actual development may not be involved in this development, will cause too much time wasted.

Technology will eventually become obsolete, but what is not obsolete is the ability to quickly learn and use new technologies.
