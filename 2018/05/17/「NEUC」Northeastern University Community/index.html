「NEUC」Northeastern University Community

Posted on 2018-05-17 Edited on 2021-11-01

Description

We propose to develop a platform for college students to engage in academic communication and resource sharing. By providing a file sharing platform where students can share their learning materials and exchange their learning experiences, these learning materials can be utilized by students throughout the school. This saves students’ time in collecting materials and maximizes the use of learning materials.

Usecases Figure



Screenshot









Background

  • For some college students, access to study materials on campus is limited, and it is difficult to search for study materials related to their majors in the Internet. When students who have these learning materials are willing to share them, there is no convenient and small-scale sharing platform.
  • When students consider changing their majors, they may need course materials of other majors, but it is difficult for students to get the latest course materials.
  • When students collect exam-related materials, they often obtain these materials through the Internet and social connections. However, time-honored examination materials are usually difficult to obtain; at the same time, most of the examination materials on the Internet are distributed in qq groups, posting bars, and WeChat public numbers, and few people integrate them. Therefore, collecting exam materials may take up a lot of review time of exam students.

Technology

Using the default framework structure of WeChat applets at the front end, the view layer is realized by wxml display page and wxss style configuration, the logic layer is realized by JavaScript files, json files for page configuration and custom component invocation.

In the back-end using the applet cloud development framework, which greatly simplifies the deployment of our back-end server, using the storage and database it provides to process and save data, and using cloud functions to write server-side logic to respond and process applet requests in a timely manner.

Project demand analysis

Phase I : Interview and exchange with university students and understand the actual needs of the users, and determine the general direction of their needs. During the discussion, we will guide the users to think about the possible functional and non-functional requirements of the system to be developed.

Phase II : We will further refine it in the form of a questionnaire, and discuss with the users about the details of the project requirements. For example, how do college students usually get their materials? Are they happy to share their study materials? How many people in the university should be able to use the system? We will finalize a series of questions related to the system requirements with the users in the second phase of the requirements analysis.

Phase III : We will make a rapid prototype and then discuss with the user about specific details, including the transformation of the interface when using the software, how to implement the various functions of the software. And the general design of the user’s graphical interface.

Glossary

Glossary of Terms
User: A general term for all users of the application.
Poster: The user who posted the resource/question in the resource/question details.
Viewer: The user who did not post the resource/question in the resource/question details.
Courseware: Learning material files that serve a pedagogical purpose for the course in question.
Test Papers: Files of various test questions for the course.
Notes: Files of self-summarized knowledge points for the course.
Resources: A generic term for courseware, test papers, and notes.
Question: A question posted by a user about a specific course.
Questions and Answers: Questions posted by users after they have successfully posted.
Title: The focus of the resource/question.
Content: A short description of the resource/question.
Category: Specifies which of the courseware, papers, or notes the uploaded resource belongs to.
Resource Area: The interface where the lesson, paper, and notes are displayed.
Resource Details: The screen that shows the details of the lesson/paper/notes.
Question and Answer Area: The interface that displays questions.
Question and answer details: The interface that shows the details of the question.
Mine: The interface that displays personal information about the current user.
Like: Add one to the like value of the current item.
Comment: Give your opinion about the current resource/issue.
Favorite: Adds the current resource/issue to the current user’s favorites.
Post: Creates new content for the resource/issue.
Message: Notification when the resource/issue posted by the current user is commented on by other users.
Grade: The grade level of the current user.
Major: The current user’s major.
Nickname: The title used by the current user in the application.
